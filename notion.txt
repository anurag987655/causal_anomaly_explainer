# Pipeline Trace: `src/run_full_pipeline.py`

Execution order:
1. `prepare_baseline.py`
2. `mixed_data.py`
3. `build_dag.py`
4. `anamoly_scoring.py`
5. `root_cause_analysis.py`
6. `behavioral_clustering.py`

## 1) `src/ingestion/prepare_baseline.py`
- Function called: `prepare_baseline()`
- Input:
  - `data/raw/Monday-WorkingHours.pcap_ISCX.csv`
  - Reads first `100000` rows (`.head(100000)`).
- Output:
  - `data/processed/baseline_training.csv`
- What it produces/why:
  - Baseline-only session-event table (BENIGN fallback label).
  - This is the baseline training dataset for DAG learning.

Example: `prepare_baseline.py` produces baseline session CSV (`baseline_training.csv`), used to train DAG.

## 2) `src/ingestion/mixed_data.py`
- Function called: `load_and_transform_combined()`
- Inputs (from `data/raw`):
  - `Thursday-WorkingHours-Morning-WebAttacks.pcap_ISCX.csv` (label hint `WebAttack`, first `50000` rows)
  - `Tuesday-WorkingHours.pcap_ISCX.csv` (label hint `BENIGN`, first `50000` rows)
- Output:
  - `data/processed/all_sessions_detailed.csv`
- What it produces/why:
  - Mixed benign + attack session-event table used for downstream scoring and explanation.

## 3) `src/casual/build_dag.py`
- Function called: `main(train_csv="data/processed/baseline_training.csv")`
- Input in this pipeline run:
  - `data/processed/baseline_training.csv` (baseline-only)
- Outputs:
  - `data/processed/dag_model_complete.pkl`
  - `data/processed/dag_edges_complete.csv`
  - `data/processed/session_features_for_clustering.csv`
  - `data/processed/feature_summary.json`
  - `data/processed/dag_graph_complete.png`
  - `data/processed/baseline_features.csv` (saved when `train_csv` is passed)
- What it produces/why:
  - Learns transition probabilities/confidence from baseline sessions.
  - Produces baseline feature matrix (`baseline_features.csv`) for statistical anomaly model training.

## 4) `src/casual/anamoly_scoring.py`
- Function called: `run_anomaly_scoring_with_rca(...)`
- Inputs:
  - `sessions_csv="data/processed/all_sessions_detailed.csv"`
  - `train_features="data/processed/baseline_features.csv"`
  - `dag_model="data/processed/dag_model_complete.pkl"`
- Output:
  - `data/processed/anomaly_scores_with_features.csv`
- What it produces/why:
  - Computes per-session hybrid anomaly scores (`anomaly_score_0_100`).
  - Flags anomalies via percentile threshold (`anomaly_percentile`, default 5% => top 5% marked `is_anomaly=True`).

## 5) `src/casual/root_cause_analysis.py`
- Function called: `run_rca_pipeline(...)`
- Inputs:
  - `sessions_csv="data/processed/all_sessions_detailed.csv"`
  - `scores_csv="data/processed/anomaly_scores_with_features.csv"`
  - `dag_model="data/processed/dag_model_complete.pkl"`
  - `limit=200`
- Outputs:
  - `data/processed/rca_explanations.csv`
  - `data/processed/rca_explanations.json`
  - `data/processed/rca_summary.txt`
  - `data/processed/rca_cluster_hints.csv`
- What it produces/why:
  - Generates RCA only for anomalous sessions (`is_anomaly == True`), up to `limit` (200 here).

## 6) `src/casual/behavioral_clustering.py`
- Function called: `run_clustering_pipeline(...)`
- Input:
  - `rca_csv="data/processed/rca_explanations.csv"`
- Outputs:
  - `data/processed/clustering_results.csv`
  - `data/processed/cluster_interpretations.json`
  - `data/processed/clustering_summary.txt`
  - `data/processed/clusterer_model.pkl`
  - `data/processed/cluster_visualization.png` (because `visualize=True`)
  - Optional: `data/processed/active_campaigns.csv` (if clusters with >=5 sessions exist)
- What it produces/why:
  - Groups explained anomalies into behavior families/campaigns via HDBSCAN.

---

## Where data splits happen (train / validation / test)

### Train split (explicit)
- Baseline training set is separated by source and used for model fitting:
  - `prepare_baseline.py` -> `baseline_training.csv`
  - `build_dag.py` trains DAG on `baseline_training.csv`
  - `anamoly_scoring.py` trains IsolationForest using `baseline_features.csv` (derived from baseline training set)

### Inference/eval set (explicit)
- Mixed dataset is separate from training and used for scoring + RCA + clustering:
  - `mixed_data.py` -> `all_sessions_detailed.csv`

### Selection splits (post-scoring filters)
- Anomaly selection split in scoring:
  - top percentile by score (`anomaly_percentile`, default 5%) becomes `is_anomaly=True`
- RCA selection split:
  - RCA processes only rows where `is_anomaly=True`, then truncates to `limit=200`

### Validation/test split
- There is **no explicit validation/test split** inside `run_full_pipeline.py` and these six called modules.
- Validation/test logic exists in separate evaluation code paths, not in this full pipeline execution path.
